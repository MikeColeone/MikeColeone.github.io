📌 总结

类型 说明 适用场景
feat 新功能 添加新模块、新 API
fix 修复 Bug 解决错误、修复崩溃问题
docs 文档变更 更新 README、修改 API 文档
style 代码格式 修改缩进、空格、样式等
perf 性能优化 提高代码执行效率
test 测试相关 添加或修改单元测试
chore 构建/工具 CI/CD 配置、依赖升级
refactor 代码重构 结构优化、提高可读性
revert 回滚 还原错误提交

如果你是用 GitHub Actions 自动部署，可以结合这些前缀触发不同的 CI/CD 任务，比如：

  • feat 触发 前端自动构建
  • fix 触发 紧急补丁部署
  • chore 不触发任何构建

如果你的团队使用 Conventional Commits 规范 或 Git Flow，建议严格按照这个格式来写提交信息！💡
