  1. flex布局

    • 布局的传统解决方案，基于盒状模型，依赖 display属性 + position属性 + float属性。它对于那些特殊布局非常不方便，比如，垂直居中就不容易实现。

    • flex布局是弹性布局，为盒装模型提供了最大的灵活性 任何容器都可以声明弹性布局 块级元素 行内元素 都可以是声明为flex布局

    • flex容器和flex项目 容器默认存在两个轴 一个主轴 一个交叉轴 主轴的开始位置main end,交叉轴成为cross main

    • 六个属性设置在容器上

      • flex-direction 决定主轴方向 row | row-reverse | column | column-reverse

      • flex-wrap 一条轴线是否换行 nowrap wrap-reverse:换行，且第一行在下方 不换行就自动调整

      • flex-flow 上面两个属性的缩写 ，默认不换行 沿着主轴排序

      • justify-content(主轴对齐方式) flex-start | flex-end |center | space-between | space-arround

      • align-items(交叉轴对齐方式) flex-start | flex=end | center | baseline(项目的的第一行文字对齐) | stretch

    • 六个属性在项目上

      • order 定义项目的排列顺序 数值越小排列越靠前 默认值为0

      • flex: flex-grow(按照剩余空间放大 默认0 ) flex-shrink(按照剩余空间等比缩小 默认1) flex-basis

      • align-self: 项目交叉轴自己的对齐方式

  9. 发异步请求都有哪些方法?比如我想发一个业务请求, 用什么方法?

    • jquery提供的异步请求方法，可以发送get,post请求，他的方法的参数是一个对象，对象有一些属性，比如说url, 发送请求的类型，get post之类，然后处理成功或者失败的回调函数（被传为另一个函数的函数）

    • fetch API是现代请求的一个api返回一个promise对象，发送一个http请求 并处理相应，fetch这个方法是有两个参数，参数是发送请求的路径，和一个对象，对象包括请求头，请求头来声明请求体的格式Content-type，比如json格式，纯文本格式等等,然后调用then来处理fetch返回的promise对象的结果，再处理返回的数据 然后通过catch捕获到错误，then会返回一个promise,catch来捕获任何可能发生的错误，超时，网络连接失败，json解析问题，拿到错误怎么处理 可以显示信息，或者记录到日志打印出来，方便我们开发的时候调试

  16. 原型链 原型

    • 每个对象身上都有一个__proto__,表示该对象的原型，__proto__也是一个对象，所以他也有自己的原型__proto__，当访问一个对象身上的方法和属性时，会在对象本身寻找，如果找不到会沿着原型的__proto__继续向上寻找，直到找到object的null。原型链是每个类，构造函数都有一个prototype表示这个类的显式原型，所创建出来的
    • 实例对象的__proto__都指向这个类的prototype，prototype也是对象，也有它的__proto__,通过修改类的显式原型prototype的__proto__指向另一个类，那么这个类创造的实例对象就可以访问到另一个类身上的属性和方法

  20. 事件循环机制

    • 是js实现异步编程的核心机制，负责协调任务队列，异步操作以及回调函数，js使用任务队列管理异步任务，避免阻塞主线程。

      • 事件循环的大概步骤：

        • 运行主线程同步任务 初始化代码执行成功完成后检查任务队列

        • 检查宏任务队列 取出第一个任务执行：什么是宏任务呢，比如setTimeout(回调函数，间隔等待时间),setInterval，，Ajax完成的回调

        • 检查微任务队列，执行所有微任务队列 什么是为任务呢 Promise的then catch finally （微服务优先级更高）

        • 更新渲染

        • 重复循环

    4. JS为什么是单线程

      • 最初的js是为了操作DOM，浏览器DOM不是线程安全的，单线程避免复杂的并发问题

      • 通过事件循环处理异步任务

      • JavaScript 本身是单线程的，不是线程安全的语言。
        因为它没有内置多线程操作下的同步机制（如锁、原子操作等），也就意味着：

        如果多个线程同时访问同一份数据，没有机制保障“谁先谁后、如何冲突处理”，就会有 数据竞争 或 状态不一致 的风险。

      • 如果多个线程同时操作一个按钮的位置、文字，结果会混乱。所以为了解决这个问题，浏览器决定：

        干脆让 JavaScript 只能单线程运行，一切按顺序来，避免并发冲突。

  27. 前端开发

    1. 页面从 URL 到页面渲染的过程

      • 输入URL之后，首先DNS解析，将域名解析为IP,建立TCP连接，也就是三次握手，之后发送HTTP请求，等待服务器响应，返回请求的资源，浏览器解析HTML,构建DOM树，加载CSS构建CSSOM树，并合并为RenderTree,然后计算元素位置，像素填充，最后合成和渲染

    2. TCP和UDP的区别

      1. TCP UDP
        连接 面向连接（三次握手） 无连接
        可靠性 可靠，数据顺序一致 不可靠，可能丢包
        速度 慢 快
        应用 HTTP、FTP、邮件 视频直播、DNS、VoIP
