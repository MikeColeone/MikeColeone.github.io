1. 基本语法

正则表达式由普通字符和元字符组成，用来匹配字符串中的特定内容。

普通字符

普通字符会原样匹配。
示例：abc
匹配字符串 "abc"。

元字符（特殊字符）

元字符具有特殊含义，常用的元字符包括：

字符 含义 示例 说明
. 匹配任意一个字符（除换行符） a.b -> “acb” . 表示任意一个字符。
\ 转义符，转义特殊字符 \. -> . \ 取消特殊字符的含义。
^ 匹配字符串的开始位置 ^abc -> “abcdef” 必须以 abc 开头。
$ 匹配字符串的结束位置 abc$ -> “xabc” 必须以 abc 结尾。
* 匹配前面字符0次或多次 ab* -> “a”, “ab” b 可以出现 0 次或多次。
+ 匹配前面字符1次或多次 ab+ -> “ab” b 至少出现 1 次。
? 匹配前面字符0次或1次 ab? -> “a”, “ab” b 可以出现 0 次或 1 次。
{n} 匹配前面字符n次 a{3} -> “aaa” a 必须连续出现 3 次。
{n,} 匹配前面字符至少 n 次 a{2,} -> “aa” a 至少出现 2 次。
{n,m} 匹配前面字符n 到 m 次 a{2,4} -> “aaa” a 最少 2 次，最多 4 次。
[] 匹配字符集合中的任意一个字符 [abc] -> “a” 匹配 a、b 或 c 中的一个字符。
` ` 或，匹配左边或右边的表达式 `a
() 分组，将多个字符视为一个整体 (ab)+ -> “abab” 整体 ab 重复多次。

2. 字符类

字符类是用来匹配某一类字符的简写。

字符类 含义 示例 说明
\d 匹配任意数字（0-9） \d -> “3” 等价于 [0-9]。
\D 匹配非数字字符 \D -> “a” 等价于 [^0-9]。
\w 匹配字母、数字、下划线 \w -> “a”, “1” 等价于 [a-zA-Z0-9_]。
\W 匹配非字母、数字、下划线 \W -> “@” 等价于 [^a-zA-Z0-9_]。
\s 匹配空白字符 \s -> “ “ 包括空格、Tab、换行等。
\S 匹配非空白字符 \S -> “a” 与 \s 相反。

3. 边界匹配

边界匹配用于限定字符串的位置。

边界符 含义 示例 说明
^ 匹配字符串的开头 ^abc -> “abcdef” 必须以 abc 开头。
$ 匹配字符串的结尾 abc$ -> “xabc” 必须以 abc 结尾。
\b 匹配单词边界 \bword\b 匹配独立的 word，不匹配 sword。
\B 匹配非单词边界 \Bword\B word 不能是单词的开头或结尾。

4. 断言

断言用于检查指定位置左右的字符，但不会消耗字符。

正向断言 (?=...)

  • 含义：当前位置右侧满足指定条件。
  • 示例：a(?=b)
    • 匹配 "a"，但要求右侧紧跟 "b"。
    • 匹配 "ab" 中的 a。

负向断言 (?!...)

  • 含义：当前位置右侧不满足指定条件。
  • 示例：a(?!b)
    • 匹配 "a"，但右侧不能是 "b"。
    • 匹配 "ac" 中的 a。

5. 分组和引用

分组 ()

  • 用于将多个字符作为一个整体进行匹配。

示例：

  • 匹配 "ac" 或 "bc"。

反向引用

  • 使用 \n 引用第 n 个分组的内容。

示例：

  • 匹配连续相同的数字，如 "11"。

6. 贪婪与懒惰匹配

贪婪匹配（默认）

  • 尽可能多地匹配字符。

示例：

  • 匹配 "a123b456b" 中的 "a123b456b"。

懒惰匹配 ?

  • 在量词后添加 ? 变为懒惰匹配，匹配尽可能少的字符。

示例：

  • 匹配 "a123b456b" 中的 "a123b"。

7. 修饰符

修饰符控制正则表达式的匹配行为：

修饰符 含义 示例
i 忽略大小写匹配 /abc/i -> “ABC”
g 全局匹配 /a/g -> 所有 a
m 多行匹配模式 ^abc -> 多行开头

示例总结

1. 匹配字符串中所有数字

正则：\d+
示例：123abc456 -> 123 和 456。

2. 去除字符串中的前导零

正则：^0+(?!\.)
示例：000123 -> 123。

3. 匹配带千分位格式的数字

正则：\\B(?=(\\d{3})+(?!\\d))
示例：1234567 -> 1,234,567。
