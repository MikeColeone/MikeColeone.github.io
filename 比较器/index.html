比较器

在比较器 Comparator 中，返回 负数、正数 和 0 的含义是用于表示排序的优先级。负数并不代表数值大小，而是决定元素的相对排序顺序。

比较器的返回值含义

  1. 返回负数 (< 0)
    表示第一个参数 (a) 应排在第二个参数 (b) 前面。

  2. 返回正数 (> 0)
    表示第二个参数 (b) 应排在第一个参数 (a) 前面。

  3. 返回 0
    表示 a 和 b 排序顺序不变（两者相等）。

复数/正数的“数值大小”不影响优先级

在比较器的实现中，返回值只看 正负号，而不关心具体数值的大小。

示例：

假设 a = 2 和 b = 5：

  • 计算 b - a，结果为 3（正数）。
    这表示 b（5）排在 a（2）前面。

再假设 a = 5 和 b = 2：

  • 计算 b - a，结果为 -3（负数）。
    这表示 a（5）排在 b（2）前面。

无论结果是 3 还是 -3，比较器只关心 正负号，不会因为数值是 3 而有更高的“优先级”。

总结

在比较器中：

  • 返回值的 正负号 决定排序顺序。
  • 返回值的数值（如 3 和 100）不会影响排序的优先级。
  • 只要是正数，就表示第二个参数优先；只要是负数，就表示第一个参数优先。

因此，复数并不表示优先级更高，它只是表示参数的顺序关系。
